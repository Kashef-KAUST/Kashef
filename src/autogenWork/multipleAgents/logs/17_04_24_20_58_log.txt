from selenium import webdriver

from selenium.webdriver.chrome.service import Service

### Python Code Version 4: Handle Unexpected Alert

Given the `UnexpectedAlertPresentException` encountered in the previous execution, it's clear that an unexpected alert popped up when attempting to book a ticket, specifically indicating that the user needs to log in first. This version of the code will handle the unexpected alert by accepting it before proceeding to interact with the login modal.

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.chrome.service import Service
from selenium.webdriver.common.alert import Alert
from webdriver_manager.chrome import ChromeDriverManager
import time

# Initialize the Chrome driver
driver = webdriver.Chrome(service=Service(ChromeDriverManager().install()))

# Navigate to the website
driver.get("http://139.196.152.44:32677/")

# Allow time for the page to load
time.sleep(5)

# Set the date
date_input = driver.find_element(By.ID, "travel_booking_date")
date_input.clear()
date_input.send_keys("08-05-2024")

# Click the "Search" button
search_button = driver.find_element(By.ID, "travel_searching_button")
search_button.click()

# Allow time for search results to load
time.sleep(5)

# Click the "Booking" button for the first available ticket
booking_button = driver.find_element(By.CLASS_NAME, "ticket_booking_button")
booking_button.click()

# Handle the unexpected alert
try:
    alert = Alert(driver)
    alert_text = alert.text
    print(f"Alert present with text: {alert_text}")
    alert.accept()  # Accept the alert
    print("Alert accepted.")
except:
    print("No alert present.")

# Allow time for any actions following the alert
time.sleep(5)

# Print the HTML content of the page after interaction
print(driver.page_source)

# Close the driver
driver.quit()
```

This code snippet includes handling for the unexpected alert that appears when attempting to book a ticket without being logged in. By accepting the alert, we can proceed with the flow, assuming the next steps would involve dealing with the login modal. However, since the task does not provide credentials or a method to bypass the captcha, this code serves to demonstrate how to handle such alerts in Selenium.
